Visión General del Curso

Este syllabus está diseñado específicamente para estudiantes de estadística con sólidos conocimientos matemáticos pero sin experiencia en ingeniería de softwareDiseño, desarrollo y mantenimiento de aplicaciones de software. El objetivo es transformar scripts de análisis aislados en productos de datos desplegables y consumibles, utilizando FastAPIFramework moderno de Python para crear APIs de alta performance como eje central y desplegando en infraestructura moderna sin servidor.

Proyecto Final: API de Datos con IA

A lo largo del semestre, construirás paso a paso una API REST completa que sirva predicciones de un modelo de Machine Learning. Partirás desde un simple script de análisis en Python y terminarás con un producto de datos profesional desplegado en la nube, accesible desde cualquier parte del mundo.

Objetivos del Proyecto

1

Transformar un análisis estadístico existente en una API web funcional usando FastAPI y Python 3.11.9.

2

Integrar un modelo de ML (Scikit-Learn o TensorFlow) que realice predicciones en tiempo real.

3

Containerizar la solución con Docker usando la imagen python:3.11.9-slim-bookworm (Debian 12).

4

Desplegar la API en Railway o Render con documentación Swagger completa y acceso público.

Fase 1 Semanas 1-4

Fundamentos

Configurar entorno local con venv, seleccionar dataset, crear esquemas Pydantic y primera versión de la API con FastAPI.

Fase 2 Semanas 5-10

API + Docker

Validación avanzada, persistencia, testing, containerización con Docker y primer despliegue en la nube.

Fase 3 Semanas 11-16

IA + Producción

Integrar modelo ML real, procesamiento asíncrono, seguridad JWT, documentación final y demo en vivo.

Evaluación Proyecto Final

1. Arquitectura API (40%)

Implementación correcta de verbos REST, códigos de estado HTTP, manejo de excepciones (try/except) y validación de datos con Pydantic. Estructura de proyecto modular.

2. Reproducibilidad de IA (30%)

El modelo debe cargar eficientemente (singleton pattern), manejar la inferencia sin bloquear el servidor, y los resultados deben ser consistentes con el entrenamiento.

3. Despliegue & Documentación (30%)

API funcional en Railway/Render (acceso público). Documentación Swagger/Redoc completa con ejemplos de request/response y autenticación básica funcional.
